
# 1. git init
- Descripción: Crea un nuevo repositorio Git.
- Caso de uso: Iniciar el control de versiones en un proyecto nuevo.
- Ejemplo:git init

# 2. git clone
- Descripción: Copia un repositorio remoto a tu computadora.
- Caso de uso: Descargar un proyecto existente.
- Ejemplo:git clone https://github.com/usuario/proyecto.git

# 3. git status
- Descripción: Muestra el estado actual del repositorio.
- Caso de uso: Ver archivos modificados o pendientes de guardar.
- Ejemplo:git status

# 4. git add
- Descripción: Agrega archivos al área de preparación.
- Caso de uso: Seleccionar archivos para el próximo commit.
- Ejemplo:git add archivo.txt

# 5. git commit
- Descripción: Guarda los cambios en el historial.
- Caso de uso: Registrar una nueva versión del proyecto.
- Ejemplo:git commit -m "Actualización del proyecto"

# 6. git log
- Descripción: Muestra el historial de commits.
- Caso de uso: Revisar cambios realizados anteriormente.
- Ejemplo:git log

# 7. git diff
- Descripción: Muestra diferencias entre versiones.
- Caso de uso: Revisar cambios antes de guardar.
- Ejemplo:git diff

# 8. git branch
- Descripción: Crea o muestra ramas.
- Caso de uso: Trabajar en nuevas funciones sin afectar el proyecto principal.
- Ejemplo:git branch desarrollo

# 9. git checkout
- Descripción: Cambia de rama.
- Caso de uso: Moverse entre diferentes versiones del proyecto.
- Ejemplo:git checkout desarrollo

# 10. git switch
- Descripción: Cambia de rama de forma más sencilla.
- Caso de uso: Navegar entre ramas.
- Ejemplo:git switch desarrollo

# 11. git merge
- Descripción: Combina ramas.
- Caso de uso: Integrar cambios de una rama a otra.
- Ejemplo:git merge desarrollo

# 12. git pull
- Descripción: Descarga y fusiona cambios remotos.
- Caso de uso: Actualizar el repositorio local.
- Ejemplo:git pull origin main

# 13. git push
- Descripción: Envía cambios al repositorio remoto.
- Caso de uso: Compartir avances con el equipo.
- Ejemplo:git push origin main

# 14. git fetch
- Descripción: Descarga cambios remotos sin fusionarlos.
- Caso de uso: Revisar actualizaciones antes de aplicarlas.
- Ejemplo:git fetch

# 15. git remote
- Descripción: Muestra los repositorios remotos configurados.
- Caso de uso: Verificar conexiones remotas.
- Ejemplo:git remote -v

# 16. git reset
- Descripción: Deshace cambios o elimina archivos del área de preparación.
- Caso de uso: Corregir errores antes de hacer commit.
- Ejemplo:git reset archivo.txt

# 17. git revert
- Descripción: Revierte un commit creando uno nuevo.
- Caso de uso: Deshacer cambios sin alterar el historial.
- Ejemplo:git revert a1b2c3

# 18. git stash
- Descripción: Guarda cambios temporalmente.
- Caso de uso: Cambiar de tarea sin perder trabajo.
- Ejemplo:git stash

# 19. git tag
- Descripción: Crea etiquetas para versiones.
- Caso de uso: Marcar lanzamientos importantes.
- Ejemplo:git tag v1.0

# 20. git rm
- Descripción: Elimina archivos del repositorio.
- Caso de uso: Borrar archivos que ya no son necesarios.
- Ejemplo:git rm archivo.txt
